# Calibrate（LED 网格校正）UX 交互规范

> 本文档是 Calibrate 板块 LED 网格校正部分的 **UX 真相源**，用于后续转换为 Claude Design 可执行 prompt。
> **只描述交互逻辑与信息架构，不包含任何视觉方向内容**（字体、颜色值、主题、圆角、阴影等一律不出现；涉及状态区分时只声明"需要可区分的状态通道"，具体视觉由 Claude Design 统一规格）。

---

## 0. 范围与原则

### 0.1 范围

**覆盖**：LED 网格校正全流程——屏幕建模 → 测试图 → 测量采集 → 标定 → 重建 → 比对 → 历史/融合/导出。

**不覆盖（维持现状，另行设计）**：
- 镜头校正（lens）页及其实时采集流程；
- AR 段（`calStageType='ar'`）整套 IA；
- 采集设置（Profile）的字段定义沿用现状，仅重新安排其入口位置（见 §6.4.4）。

### 0.2 术语与去标识化（强约束）

| 规范用语 | 含义 | 旧内部代号（**禁止出现在任何 UI 文案**） |
|---|---|---|
| **全站仪导入** | 用全站仪实测点位 CSV 导入测量数据 | M1 |
| **视觉校正** | 屏幕显示测试图 + 摄影机拍摄 + 光束法平差重建 | M2 |
| **融合** | 全站仪锚定 + 视觉稠密化的合并求解 | M1+M2 融合 |

其余 UI 术语约定：
- 「屏幕」= 一块 LED 屏（若干箱体拼成）；「箱体」= cabinet；「舞台」= 项目内所有屏幕构成的三维空间。
- 「原始网格」= 由屏幕设计参数生成的理想模型；「新建网格」= 重建产出的实测模型。
- 「测试图」= ChArUco 校正图案（屏幕显示用）。

### 0.3 设计原则

1. **单一工作区**：网格校正的全部操作发生在一个页面里，页面主体是三维视口；操作与设置通过工具栏、弹出面板与检查器（Inspector）完成，不做多页跳转。
2. **引导式流程**：需要用户做类型选择时（全站仪 vs 视觉校正），先给大而明确的选择卡，选定后进入对应流程；流程内**随时可切换类型**，切换入口常驻。
3. **表单即所见**：屏幕建模是参数化表单驱动，任何参数改动即时反映在三维视口中（不需要"应用"按钮）。
4. **检查器承载细节**：主流程只暴露必要决策；所有细节选项（内参文件、重建偏好、显示设置等）收进检查器的分组列表，可忽略、有合理默认值。
5. **状态三通道**：任何状态区分（成功/警告/失败、实测/推测/外推）必须同时具备图形符号 + 文字标注 + 视觉通道三种表达（具体视觉表达由设计稿定义）。

## 1. IA 总览

板块内共 **2 个页面** + 若干弹层：

```
Calibrate（LED）
│
├── ① 概览页（项目入口 / 多项目状态）
│      └─ 打开某项目 → 进入 ②
│
└── ② 网格校正工作区（单页，三维视口为主体）
       ├─ 常驻区域：顶部工具栏 / 左侧大纲 / 中央三维视口 / 右侧检查器 / 底部状态栏
       ├─ 弹层：测量类型选择器（全站仪 vs 视觉校正）
       ├─ 停靠面板：测量导入流程面板（全站仪流 / 视觉校正流，左上角带类型切换）
       ├─ 弹层：实时采集对话框（二级，接入现场摄影机后）
       ├─ 弹层：新建屏幕菜单 / 导出对话框 / 融合对话框 / 指导卡预览
       └─ 弹层：长任务进度（重建 / 离线批处理，可取消）
```

原「屏幕与设计 / 测量导入 / 重建与预览 / 历史与导出」四个左栏页面**全部取消**，其功能按下文分布进工作区的各区域。

## 2. 概览页

定位：项目入口与多项目健康总览。轻量，不承载任何校正操作。

### 2.1 空态（未打开项目）

居中排布三组内容：
- **主操作**：「打开项目」（系统文件选择器，选 project.yaml）与「新建项目」两个并列主按钮。
- **最近的项目**列表：每行 = 项目名 + 路径 + 屏幕数 + 最近一次重建状态摘要；单击整行即打开；行尾提供「从列表移除」次级操作（不删除磁盘文件，需悬停或聚焦才出现）。
- 列表为空时显示一句引导文案（说明打开/新建即可开始）。

### 2.2 已打开态

- **页头**：当前项目名 + 路径；右侧主按钮「进入工作区」；次级操作「关闭项目」（回到空态）。
- **项目切换器**：下拉列出最近项目，选中即原地切换（不经过空态）。
- **项目状态表**：对最近项目逐项聚合，列 = 项目 / 屏幕数 / 网格状态（未重建 · 已重建 · 已导出，按该项目最近一次重建 run 派生）/ 最近活动时间。单击行 = 切换到该项目并进入工作区。
- 本页**不再显示镜头状态列**（镜头校正不在本板块范围内，且无可靠状态信号）。

## 3. 工作区布局

一屏五区。除三维视口外，左右两栏均可折叠（折叠后留有把手可展开），给视口让出最大空间。

```
┌────────────────────────────────────────────────────────────┐
│ 顶部工具栏：项目/屏幕上下文 ｜ 阶段动作 ｜ 视图控制          │
├──────────┬──────────────────────────────────┬──────────────┤
│ 左侧大纲  │        中央三维视口               │ 右侧检查器    │
│ (场景树 + │   (LED 舞台空间；叠加控件见 §3.3) │ (上下文敏感)  │
│  数据源)  │                                  │              │
├──────────┴──────────────────────────────────┴──────────────┤
│ 底部状态栏：单位/坐标读数 ｜ 后台任务指示 ｜ 提示信息        │
└────────────────────────────────────────────────────────────┘
```

### 3.1 顶部工具栏

从左到右三组：

1. **上下文组**：返回概览按钮 · 当前项目名（点击展开项目切换下拉）· 当前屏幕选择器（多屏时）。
2. **阶段动作组**（网格校正主流程的四个动作按钮，始终可见，按前置条件启用/禁用，禁用时悬停提示缺什么）：
   - 「测试图」→ 生成/管理测试图（§5）；
   - 「测量导入」→ 打开测量类型选择器（§6.1）；
   - 「重建」→ 触发重建（§7.2；无测量数据时禁用）；
   - 「导出」→ 打开导出对话框（§9.3；无重建结果时禁用）。
3. **视图控制组**：视图预设（正视/顶视/侧视/自由）· 聚焦选中（快捷键 F）· 复位视角（Home）· 显示开关菜单（§3.3.2）。

### 3.2 中央三维视口

**渲染内容**：地面网格 + 世界坐标轴指示（含轴向朝向小组件，点击轴头可跳正交视角）+ 项目内全部屏幕模型 + 按显示开关叠加的测量点 / 新建网格 / 测试图纹理。

**相机交互**（与主流三维软件一致）：
- 左键拖拽 = 环绕（orbit）；右键或中键拖拽 = 平移；滚轮 = 缩放；
- F = 聚焦当前选中对象；Home = 框住整个舞台；双击对象 = 聚焦该对象。

**拾取与选择**（两级模式，类似 Blender 的 Object/Edit 模式）：
- **对象模式**（默认）：单击选中整块屏幕（描边高亮），Esc 取消；选中屏幕后检查器显示屏幕参数（§4.2）。
- **箱体模式**：在选中屏幕上进入（快捷键 Tab，或检查器内「编辑箱体」按钮）；此时拾取粒度变为单个箱体/箱体角点，支持单击、Shift 加选、框选；用于遮罩与参考点标注（§4.4）。退出（Tab/Esc）回对象模式。
- 空白处单击 = 清空选择，检查器回落到全局「校正细节选项」（§3.4.3）。

**对象操作**：选中屏幕出现移动/旋转操纵器（gizmo），可在舞台内摆放屏幕位置与朝向（用于多屏舞台布局，如主墙 + 地屏）；数值精调在检查器的「变换」分组里。

### 3.3 视口叠加控件

叠加在三维视口四角，不占布局空间：

#### 3.3.1 左上：流程上下文
- 平时显示当前屏幕名 + 当前模式（对象/箱体）。
- 测量导入流程激活时，此处变为**测量类型切换入口**：显示当前类型（「全站仪导入」或「视觉校正」），点击弹出切换菜单，随时换到另一类型（详见 §6.1.3）。

#### 3.3.2 右上：显示开关（也可从工具栏视图菜单打开）
逐项开关，均可独立组合：
- 测量点（含点名标签的二级开关）；
- 测试图贴合预览（§5.3）；
- 来源着色（重建顶点 measured / interpolated / extrapolated 三态着色，§8.3）；
- 地面网格 / 坐标轴；
- 被遮罩箱体的显隐方式（镂空 or 半透明占位）。

#### 3.3.3 底部中央：网格版本切换器（§8.1）
仅当当前屏幕存在重建结果时出现：「原始网格 ⇄ 新建网格」两段式切换。

#### 3.3.4 左下：视口内提示
显示当前模式的操作提示（如箱体模式下"单击选择 · Shift 加选 · 框选"），随模式切换更新。

### 3.4 左侧大纲（场景树 + 数据源）

树形结构，节点即数据实体，选中节点驱动检查器与视口高亮：

```
项目名
└─ 屏幕 MAIN
   ├─ 设计模型            ← 选中 = 对象模式选中该屏
   ├─ 测试图              ← 未生成时显示"未生成"占位
   ├─ 测量数据
   │   ├─ 全站仪数据集（导入时间 · 点数）    ← 可多次导入，保留最新，历史入 run
   │   └─ 视觉采集会话（时间 · 张数/姿位数）
   └─ 重建结果
       ├─ Run #12（最新 · 当前显示）
       ├─ Run #11
       └─ …
```

- 节点行内提供上下文操作（悬停出现）：如测试图节点的「重新生成」、run 节点的「在视口中查看」「设为当前」「打开输出文件夹」。
- 「重建结果」分组头部提供「融合…」入口（当同屏同时存在全站仪数据与视觉结果时启用，§9.2）。
- 树支持右键菜单，内容与行内操作一致（同一能力双入口）。
- 树底部：「新建屏幕」按钮（§4.1）。

### 3.5 右侧检查器（Inspector）

上下文敏感，按当前选中对象切换内容；统一为**可折叠分组的属性列表**（标签左、控件右），分组标题可折叠、记忆折叠状态。各上下文的具体内容见对应章节：

| 选中对象 | 检查器内容 | 章节 |
|---|---|---|
| 屏幕（对象模式） | 屏幕建模参数表单 + 变换 + 派生信息 | §4.2 |
| 箱体/顶点（箱体模式） | 箱体属性、遮罩、参考点角色、多选批量操作 | §4.4 |
| 测量点 | 点名、坐标、不确定度、来源 | §6.2.3 |
| 测量数据集节点 | 导入摘要、统计、警告列表 | §6.2.3 / §6.3 |
| 重建 run | 质量指标、方法、时间、产物路径 | §8.4 |
| 无选中 | 全局「校正细节选项」 | §3.5.1 |

#### 3.5.1 全局「校正细节选项」（无选中时的默认内容）

网格校正全流程的细节配置集中在这一个列表里，全部有默认值、全部可忽略。分组：

- **标定**：
  - 相机内参来源：`自动标定（默认）` / `导入棋盘格内参文件`（文件选择 + 已选文件名回显 + 清除按钮）。这是一个小配置项：不导入则重建时自动完成自标定；导入后重建改用外部内参（§7.1）。
- **重建**：
  - 重建方法：`自动（默认，按数据情况分派）` / 手动指定（下拉：直接映射 / 径向基 / 边界插值 / 名义形状）；
  - 显示上一次实际采用的方法（只读）。
- **测量显示**：实测/推测判定阈值说明（只读展示当前 σ 阈值）；测量点标签默认显隐。
- **导出默认**：默认导出目标（Disguise / Unreal / 中性）、默认输出目录。
- **采集**：默认采集配置（Profile）选择（管理入口跳实时采集对话框内的同一管理界面，§6.4.4）。

### 3.6 底部状态栏

- 左：单位（m/mm）与指针命中处的世界坐标读数（视口悬停时实时更新）。
- 中：后台任务指示（重建/批处理运行中显示任务名 + 进度 + 点击展开进度弹层）。
- 右：轻量提示区（最近一次操作的结果回执，如"已导入 214 个测量点"，自动消隐，可点击查看详情）。

## 4. 屏幕建模

### 4.1 新建屏幕

大纲树底部「新建屏幕」→ 弹出**类型菜单**（预设 kind，选中即在舞台原点生成一块带默认参数的屏幕并选中它，随后在检查器里调参）：

- 平面墙（默认 8×3 箱体）
- 弧形墙（对称弧）
- L 形墙
- U 形墙
- 自定义分段墙
- 地屏（水平放置的平面阵列）

**无向导、无拖拽建模**——新建后的一切都靠检查器参数表单，改哪个参数视口即时重建（§4.3）。

### 4.2 屏幕参数表单（检查器 · 对象模式选中屏幕时）

分组与字段（标注 ⊙ 的字段按上游枚举值条件显隐）：

**① 标识**
| 字段 | 控件 | 说明 |
|---|---|---|
| 屏幕名 | 文本 | 唯一，用作数据关联 ID 的显示名 |
| 屏幕类别 | 只读 | 墙 / 地屏（由新建时类型决定） |

**② 箱体**
| 字段 | 控件 | 默认 |
|---|---|---|
| 箱体预设 | 下拉（厂商预设库 + 「自定义」） | 自定义 |
| 箱体尺寸 | 双数值（宽 × 高，mm） | 500 × 500 |
| 箱体像素 | 双数值（宽 × 高，px） | 192 × 192 |

选预设自动回填尺寸/像素并锁定；改回「自定义」解锁。

**③ 布局**
| 字段 | 控件 | 默认 |
|---|---|---|
| 列数 | 整数步进 | 8 |
| 行数 | 整数步进 | 3 |
| 离地高度 | 数值（m） | 0 |

**④ 形状（曲率模型）** —— 单选：`平直` / `对称弧` / `L 形` / `U 形` / `自定义分段`
- ⊙ 对称弧：`中心平直列数`（整数）+ `每列折角`（度/列）。
- ⊙ L 形：`左面列数` + `转角软化列数`（0 = 硬折）+ `转角角度`（度；正 = 凹向观众，负 = 凸向观众）。右面列数 = 总列 − 左 − 角，作为只读派生值实时显示；不合法时内联报错。
- ⊙ U 形：`每翼列数` + `每角软化列数` + `每角角度`。中段列数同上派生 + 校验。
- ⊙ 自定义分段：段列表（每段 = `列数` + `累计转角`），支持增删段、拖拽排序；内联校验「各段列数之和 = 总列数」。
- 地屏类别时本组隐藏（恒为平面）。

**⑤ 变换**
| 字段 | 控件 |
|---|---|
| 位置 | 三数值（X/Y/Z，m），与视口 gizmo 双向同步 |
| 朝向 | 数值（绕竖直轴旋转，度） |

**⑥ 派生信息（只读，实时计算）**
- 整屏物理尺寸（m）· 整屏像素画布（W×H px）· 箱体总数（含被遮罩数）· 顶点网格规模（(列+1)×(行+1)）。

**⑦ 高级**
- 形状先验补充：`弯折缝所在列`（多值，供重建拟合参考）；
- 底部补全：`最低可测行` + `补全方式` + `假定高度`（全站仪测不到底排时的推算配置）。

### 4.3 即时重建与校验

- 任何参数变更 → 视口模型即时重建（同一对象身份保留，选中态/变换不丢）。
- 非法组合**不阻塞输入**，而是：字段级内联错误 + 视口保持上一次合法状态 + 表单顶部汇总条说明未生效原因。
- 参数表单改动支持撤销/重做（Cmd/Ctrl+Z / Shift+Cmd/Ctrl+Z），与箱体编辑共用一条撤销栈（仅覆盖设计态编辑，不涉及重建等流水线动作）。
- 屏幕参数变更后若已存在测试图/测量数据/重建结果，相应大纲节点标记「已过期」（图形符号 + 文字），提示需重新生成/重建，但不自动删除旧数据。

### 4.4 箱体编辑（箱体模式）

在选中屏幕上按 Tab（或检查器「编辑箱体」）进入。视口把该屏渲染为可逐格拾取的箱体阵列，其余屏幕淡出。三个子工具（视口顶部小工具条 + 快捷键）：

1. **选择（V）**：单击/Shift 加选/框选箱体。检查器显示：单选 = 箱体行列号、遮罩开关；多选 = 选区构成统计 + 批量「设为遮罩 / 取消遮罩」。
2. **遮罩（M）**：直接点击/拖刷切换箱体遮罩（异形屏镂空）。仅屏幕为「异形」形状模式时可用；矩形模式下工具禁用并提示先在检查器切换形状模式。被遮罩箱体按显示开关设置渲染为镂空或半透明占位。
3. **参考点（R）**：拾取粒度切换为**箱体角点（顶点）**。依次为三个角色指派顶点：`原点（1）`、`X 轴方向点（2）`、`XY 平面点（3）`。已指派顶点在视口常驻标记（符号 + 角色文字）；检查器列出三角色的当前顶点名，可单独清除重指。这三点即全站仪建系的三个参考点，会印在指导卡上（§6.2.1）。

退出箱体模式自动保存编辑（遮罩、参考点角色、底部补全等随项目落盘；见附录 A 的现状差距）。

## 5. 测试图生成与贴合预览

### 5.1 入口与生成

- 入口：工具栏「测试图」按钮，或大纲树「测试图」节点。
- 点击后检查器切换到**测试图面板**：
  - 参数：图案方案（ChArUco / 密集编码点）· 屏幕标识码 · 目标屏幕（默认当前屏）；
  - 主按钮「生成测试图」；生成为短任务，按钮内联进度；
  - 完成后显示摘要：覆盖箱体数 · 标记总数 · 输出目录（可打开文件夹）。

### 5.2 生成结果的三种去向

生成完成后测试图面板提供三个并列动作：
1. **在视口中预览**（默认自动开启，即 §5.3 的贴合预览开关）；
2. **发送到播放器**：选择目标显示器（枚举列表）→ 在该显示器全屏显示拼合测试图（用于把图案真正打到 LED 处理器/屏上）；播放中此按钮变为「停止播放」；
3. **打开输出文件夹**（逐箱体 PNG + 拼合 PNG）。

### 5.3 三维贴合预览

- 开启后，拼合测试图作为纹理贴合到该屏幕模型表面，逐箱体边界与图案对位可直接检查（弧形/L/U 屏可环绕查看图案在折角处的连续性）。
- 被遮罩箱体区域不贴图。
- 屏幕参数变更导致测试图过期时，贴图上叠加「已过期」角标，提示重新生成。
- 该预览是显示开关之一（§3.3.2），与测量点、来源着色等可同时叠加。

## 6. 测量导入

### 6.1 引导式类型选择

#### 6.1.1 选择器弹层
点击工具栏「测量导入」→ 弹出居中选择器，内容为**两张大选择卡**（等宽并列，整卡可点）：

| 卡片 | 标题 | 说明文案要点 | 前置状态提示 |
|---|---|---|---|
| A | **全站仪导入** | 用全站仪实测箱体角点，毫米级精度；需要现场测量与 CSV 点位文件 | 显示是否已设参考点（§4.4-3） |
| B | **视觉校正** | 屏幕显示测试图，摄影机多角度拍摄，自动重建箱体位姿；支持离线照片与现场实时采集 | 显示测试图是否已生成 |

- 卡片下方各有一行"适合场景"辅助说明，帮助新手选择。
- 若某屏已有对应类型的数据，卡片上显示既有数据摘要（如"已有 214 点 · 6 月 30 日"），此时点击进入的是该流程的结果态而非空流程。
- Esc / 点击遮罩关闭。

#### 6.1.2 进入流程面板
选定类型后，选择器关闭，工作区左侧大纲位置被**测量导入流程面板**替换（占据同一停靠位，视口保持可见可交互——测量流程中经常需要看模型）。面板可通过头部关闭按钮退出，退回常规大纲。

#### 6.1.3 类型切换入口（常驻）
流程面板期间，**视口左上角流程上下文控件**（§3.3.1）显示当前类型名，点击展开菜单切换到另一类型：
- 切换不丢数据：各类型的流程进度独立保存，切回时恢复原进度；
- 面板头部同样显示类型名 + 切换下拉（双入口，同一行为）。

### 6.2 全站仪导入流

面板为纵向三步（步骤条显示进度，可回看已完成步骤）：

#### 6.2.1 第 1 步 · 测量准备（可跳过）
- 前置检查：参考点三角色是否已指派（未指派则给内联警告 + 「去设置参考点」跳转按钮，跳到箱体模式参考点工具）。
- 「生成指导卡」：生成给现场测量员的指导文档（含参考点位置图示与测点命名规则），弹层内预览，提供「导出 PDF」。
- 此步可显式「跳过」（已有 CSV 时直接进第 2 步）。

#### 6.2.2 第 2 步 · 导入 CSV
- 文件选择（或拖拽文件到面板）；
- 采样模式选项：`按网格（默认）` / `散点`；散点时需填测量列数；
- 「导入」→ 短任务进度 → 成功进第 3 步；失败在本步内联展示错误（文件哪一行、什么问题）。

#### 6.2.3 第 3 步 · 结果与检查
- **统计块**：实测 / 推测 / 离群 / 缺失 四个计数（每个可点击，点击 = 在点表中筛选该类）。
- **警告列表**：逐条可展开。
- **点表**：点名 / 坐标(x,y,z m) / 不确定度(mm) / 类别；单击行 = 视口中对应测量点高亮并聚焦，检查器显示点详情。
- 同时视口自动开启「测量点」显示开关：测量点以标记渲染在屏幕模型对应位置上，类别用三通道区分。
- 尾部主按钮：「重建」（等价于工具栏重建按钮，§7.2）。

### 6.3 视觉校正流

面板为纵向四步：

#### 6.3.1 第 1 步 · 测试图
- 检查当前屏测试图状态：未生成 → 内嵌「生成测试图」（同 §5.1 参数与动作）；已生成 → 摘要 + 「在视口预览」「发送到播放器」快捷键（同 §5.2）。
- 已生成即视为完成，可进下一步。

#### 6.3.2 第 2 步 · 素材采集（二选一，选项卡切换）

**选项卡 A · 离线照片**
- 「选择照片文件夹」→ 扫描该文件夹 → 摘要卡：照片张数 / 格式分布 / 可疑文件（过小、非图片）计数；
- 可重选文件夹；确认后此步完成，照片集登记为该屏的一次「视觉采集会话」（大纲树可见）。

**选项卡 B · 现场实时采集**
- 「接入摄影机…」按钮 → 打开**实时采集对话框**（二级弹层，§6.4）；
- 对话框内完成采集并关闭后，本步显示采集会话摘要（姿位数 / 张数 / 覆盖度），视为完成。

#### 6.3.3 第 3 步 · 标定（可忽略的小步骤）
- 单行摘要显示当前内参来源：`自动标定（默认）` 或 `外部内参文件：<文件名>`；
- 行尾「更改…」跳到检查器全局「校正细节选项 → 标定」（§3.5.1）；
- 无需任何操作即可继续——这一步存在的意义是让用户知道有此选项。

#### 6.3.4 第 4 步 · 重建
- 主按钮「开始重建」→ 长任务进度（§7.2 统一规格）；
- 完成后内联结果摘要：重投影 RMS(px) / 使用观测数 / 对齐残差(m) / 逐箱体质量表（行点击 = 视口高亮该箱体）；
- 尾部动作：「在视口中比对」（开启网格版本切换器并聚焦，§8）。

### 6.4 实时采集对话框（二级弹层）

模态对话框，覆盖工作区（采集期间不操作三维视口）。布局四区：

```
┌───────────────────────────────────────────────┐
│ 头部：采集配置选择 ｜ 连接状态 ｜ 关闭          │
├──────────────────────────────┬────────────────┤
│                              │ 采集进度        │
│      摄影机实时画面           │  · 姿位 n / N   │
│   （含检测反馈叠加标记）       │  · 覆盖度       │
│                              │  · 已采列表     │
├──────────────────────────────┴────────────────┤
│ 底部操作条：拍摄 ｜ 跳过此姿位 ｜ 完成 ｜ 中止  │
└───────────────────────────────────────────────┘
```

#### 6.4.1 接入与连接
- 头部选择**采集配置**（Profile，含视频后端/设备/追踪协议/端口/姿位数等，沿用现有字段），旁边「管理配置…」展开配置管理（§6.4.4）；
- 「连接」→ 连接状态三态：连接中 / 已连接（显示实时画面）/ 失败（错误说明 + 重试）。

#### 6.4.2 采集主循环
- 实时画面上叠加检测反馈（识别到的标记框 + 数量），画面下沿一行反馈文字（如"检测到 41 个标记 · 曝光偏低"）；
- 右侧进度区：当前姿位 / 总姿位、覆盖度指示（哪些区域已被足够观测）、逐姿位已采列表（可标记重拍）；
- 需要屏幕出图时（图案联动）：对话框自动通过播放器出图，状态行提示"正在显示第 n 组图案"；
- 底部操作：「拍摄」（主操作，空格键等效）· 「跳过此姿位」· 「完成」（采够即可提前结束）· 「中止」（丢弃确认后关闭）。

#### 6.4.3 结束
- 「完成」→ 汇总页（张数/姿位/覆盖度/警告）→ 「保存会话」关闭对话框，回到流程面板第 2 步的完成态。

#### 6.4.4 采集配置管理
- 对话框头部「管理配置…」展开为列表 + 表单：新建/复制/编辑/删除命名配置；字段沿用现有 Profile 定义（名称、视频后端、设备、追踪协议、端口、姿位数、稳定时间、连拍数、反相、灰码同步、输出目录）。此处是配置的唯一管理入口，检查器「校正细节选项 → 采集」只做默认配置的选择。

## 7. 标定与重建

### 7.1 内参策略（小配置项，可忽略）

- 默认**自动标定**：重建过程内联完成自标定，用户无感。
- 可选**导入外部棋盘格内参文件**：在检查器全局「校正细节选项 → 标定」中选择内参 yaml；选择后所有后续重建使用该内参，来源在重建结果中记录并回显（`内参来源：自动 / 外部文件`）。
- 不提供强制流程步骤；视觉校正流第 3 步只是该选项的一个"路标"（§6.3.3）。

### 7.2 重建执行（统一长任务规格）

触发入口：工具栏「重建」/ 全站仪流第 3 步尾部 / 视觉流第 4 步。三入口同一行为：

- 启动前置校验：当前屏是否有可用测量数据；两类数据都有时询问用哪份（单选：全站仪 / 视觉 / 融合结果，默认最新一份）。
- 运行中：进度弹层显示阶段流水（如 载入 → 检测 → 精化 → 平差 → 对齐 → 输出），当前阶段高亮，附阶段内百分比；提供「取消」（取消需确认）；弹层可最小化到底部状态栏继续后台跑。
- 完成：进度弹层变为结果摘要（质量指标 + 「在视口中查看」主按钮）；大纲树「重建结果」新增 run 节点并自动设为当前；视口自动切到「新建网格」并开启版本切换器。
- 失败：弹层展示错误详情（可复制），保留输入现场，可直接重试。

## 8. 网格比对与预览

### 8.1 网格版本切换器（核心比对交互）

视口底部中央常驻控件（有重建结果时出现）：

- 两段式开关：`原始网格` ⇄ `新建网格`；**单击即瞬时切换，无过渡动画**——设计目标就是支持高频反复点击，通过视觉残留直接比对两版网格在形状/位置/角度上的差异；
- 快捷键 `\`（反斜杠）等效切换，支持按住连击；
- 切换时相机完全不动（这是比对成立的前提）；
- 第三个辅助档 `叠加`：两版网格同时显示（当前版实体 + 另一版线框幽灵），用于静态观察偏差方向；
- 当前显示的是哪一版，除控件选中态外，视口左上角上下文区同步显示文字标注（三通道原则）。

### 8.2 比对目标选择

- 默认比对「当前 run」；大纲树中任意 run 可「设为当前」；
- 检查器 run 详情内提供「与另一 run 比对」：临时把切换器两端变为 Run A ⇄ Run B（退出后恢复 原始 ⇄ 当前）。

### 8.3 来源着色与测量点

- 显示开关「来源着色」：新建网格顶点按 实测 / 插值 / 外推 三态着色，图例常驻视口右下（图例项可点击单独隐藏某类）；旧数据无来源信息时整体标注「来源未知」，不冒充全实测；
- 显示开关「测量点」：测量点标记叠加显示，与网格切换无关（点位不随切换消失，便于对照点与面）。

### 8.4 质量指标（检查器 · 选中 run 时）

- 指标卡组：RMS / 最大偏差 / 顶点数 / 实测占比 等（与后端 QualityMetrics 对齐，逐项附一句话解释）；
- 元信息：方法（实际采用的重建器）、测量来源（全站仪/视觉/融合）、内参来源、时间、产物路径（可打开文件夹）；
- 动作：「在视口中查看」「设为当前」「与另一 run 比对」「导出…」。

## 9. 历史、融合与导出

### 9.1 历史

- 不再有独立历史页：全部 run 以大纲树「重建结果」子节点呈现（倒序，最新在上，标注"当前"）；
- run 节点选中 = 检查器详情（§8.4）；双击 = 在视口中查看；
- 节点悬停操作：查看 / 设为当前 / 打开输出文件夹；
- 数量多时分组折叠（按日期），树内可容纳全部历史。

### 9.2 融合

- 入口：大纲树「重建结果」分组头部「融合…」（仅当该屏同时存在全站仪数据与视觉结果时启用，否则禁用 + 悬停说明缺哪份）。
- 融合对话框：
  - 输入回显：全站仪数据集（当前项目的，只读）；视觉结果（下拉选一份位姿报告）；
  - 选项：`允许尺度缩放`（开关，默认关，附一句话解释何时该开）；
  - 「开始融合」→ 短任务 → 结果：锚点数 / 锚点 RMS(mm) / 尺度因子（锁定时标注）+ 逐锚点残差表（行点击 = 视口高亮该锚点）；
  - 「保存为融合结果」→ 生成融合产物，登记进大纲树（可作为重建输入源，§7.2）。

### 9.3 导出

工具栏「导出」→ 导出对话框：
- **网格 OBJ**：目标单选 `Disguise / Unreal / 中性`（各附一句差异说明，如坐标手性/顶点上限）；导出源（默认当前 run，可换）；输出路径；「导出」→ 完成态给「打开所在文件夹」；超顶点上限等错误内联展示。
- **指导卡 PDF**：与 §6.2.1 同一能力的第二入口（面向"测完回来补导"场景）。
- 导出记录追加到对应 run 的详情里（导出过什么目标、何时、到哪）。

## 10. 状态与反馈通则

- **空态**：每个区域自带空态与下一步指引——视口（无屏幕：引导新建屏幕）、大纲子节点（未生成/未导入：占位 + 行内动作）、检查器（无选中：全局选项 + 一句"选中对象查看属性"）。空态永远给出明确的单一主动作。
- **前置条件**：动作不满足前置时**禁用而非隐藏**，悬停提示缺什么、点击提示可跳转到解决处（如"先设置参考点 → 去设置"）。
- **长任务**（重建/离线批处理/融合）：统一使用 §7.2 的进度弹层规格——阶段流水 + 可取消 + 可最小化到状态栏；同屏同类任务互斥（运行中禁止重复触发）。
- **短任务**（生成测试图/导入 CSV/指导卡）：触发按钮内联进度，不弹层。
- **过期传播**：上游变更（屏幕参数）→ 下游产物（测试图/测量/重建）标「已过期」，只标注不删除；重新执行后清除标记。
- **破坏性操作**：中止采集、删除采集会话、覆盖导出文件需确认对话框，明确说明会丢什么。
- **错误**：就地展示（哪一步错就显示在哪一步），含可操作信息（文件/行号/原因），可复制全文；不用只说"失败"的全局 toast。
- **状态三通道**：所有状态分类（点类别、顶点来源、任务状态、过期标记）必须"图形符号 + 文字 + 视觉通道"三者齐备；本规范只约束该原则，具体符号与视觉表达由设计稿统一定义。
- **快捷键汇总**：V/M/R（箱体子工具）、Tab（对象⇄箱体模式）、F（聚焦）、Home（复位）、`\`（网格版本切换）、空格（采集对话框拍摄）、Esc（退选/关弹层）、Cmd/Ctrl+Z 系（撤销重做）。快捷键均有对应可点击控件，不作为唯一入口。
